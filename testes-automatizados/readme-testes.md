Tipos de teste automatizados

1 - Testes de ponta a ponta (e2e)
 - Simulam o ambiente real
 = Testam fluxos mais complexos
 -  Devem ser poucos pois são muito custosos para rodar

2 - Testes de integração
- Testam trechos de código que trabalham em conjunto
- Devem ser mais numeros que os e2e

2 - Teste unitários
- Testam uma pequena unidade de código
- Devem ser os mais constantes dentro do código
